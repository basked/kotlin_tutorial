1)Интерфейс - недокласс который не может иметь состояния(не может иметь переменных для хранения чего-то)
2)Можем иметь реализацию и функции которые необходимо реализовать в потомке
3)В интерфейсе могут быть определены функции и свойства
4)Переопределять НУЖНО методы, которые не имеют реализации и МОЖНО методы,
 которые имеют реализацию
5) Если у нескольких интерфейсов есть одинаковые методы то то при наследовании от этих двух инрефейсов будет ошибка,
    потому что не понятно какой из двух методов вызвать
    В данном случаем мы можем просто переопрделить один метод либо вызвать суперклас с нужным интерфейсом
